Projet Classic Models

Le but de ce projet est de créer un tableau de bord pour l'administration d'un site de vente de modèles réduits.

Dans un premier temps nous avons procédé à une analyse MERISE (création du MCD puis du MPD) de la base de données à la suite de la création de cette dernière en utilisant notamment l'outil Figma pour créer un support visuel pour nous aider. A la fin de cette opération les relations entre tables ont été créés à l'aide de clés primaires et étrangères pertinentes dans les tables.

Par la suite nous avons créer les requêtes SQL nécessaires à chacune des sections du site, par exemple nous avons créé des requêtes pour obtenir une section présentant les produits dont le stock est faible ou bien une section affichant les meilleurs vendeurs. Ces requêtes ont pu nécessiter parfois de nombreuses jointures (jusqu'à quatre) pour obtenir des informations que l'ont souhaitait utiliser depuis plusieurs tables ; par exemple pour obtenir la ville dans laquelle travaille un employé, celle-ci étant dans une table distincte de celle des employés.

Nous avons ensuite commencer à construire notre Back-end sous le modèle 'Modèle-Vue-Contrôleur'. Nous avons créé les fonctions nécessaires à la génération et éventuellement au calcul des données requise ; nous avons notamment écrit les fonctions permettant de faire des boucles pour afficher plusieurs éléments au sein de nos pages html par la suite ainsi que les fonctions des Modèles permettant les interactions avec la base de données à l'aide de l'objet PDO.

Finalement, nous restait à faire le templating et à intégrer dans nos pages html le code PHP nécessaire à l'appel des fonctions créées précédemment afin d'afficher sur les pages les éléments récupérés sur la base de données.
